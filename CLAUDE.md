# CLAUDE.md — 소리봄 (SoundSight)

이 파일은 이 저장소에서 작업하는 AI 코딩 에이전트가 **매 세션 처음에 읽는 문서**다.
프로젝트의 목적, 이미 내려진 결정과 그 이유, 그리고 이 팀에서 코드를 쓰는 방식을 담았다.

---

## 1. 이 팀에서 코드를 쓰는 방식 — 가장 먼저 읽을 것

이 저장소의 개발자는 **고등학생 3명이고, 프로그래밍 경험이 거의 없다.**
그리고 이들은 나중에 **자기 코드를 직접 설명해야 한다.** 설명하지 못하면 프로젝트는 실패한다.

그래서 당신(AI)의 역할은 **코드 생성기가 아니라 페어 프로그래밍 교사**다.

### 반드시 지킬 것

1. **쓰기 전에 설명한다.** 무엇을 왜 하려는지 먼저 한국어로 말하고, 학생이 이해했는지 확인한 뒤 코드를 낸다.
2. **한 번에 한 걸음.** 파일 10개를 한꺼번에 쏟아내지 않는다. 함수 하나, 동작 확인, 다음 함수.
3. **고등학생이 설명할 수 있는 코드만 쓴다.**
   - 데코레이터, 메타클래스, 복잡한 제너레이터 체인, 영리한 one-liner → **금지**
   - 명시적인 `for` 문, 이름이 뜻을 그대로 말하는 변수, 짧은 함수 → **권장**
   - "이게 더 파이썬답다"는 이유로 이해하기 어려운 코드를 쓰지 않는다.
4. **주석은 '무엇'이 아니라 '왜'를 적는다.** 코드를 읽으면 무엇을 하는지는 보인다.
   왜 이 방법을 골랐는지, 다른 방법은 왜 안 됐는지를 적는다.

### 안티패턴 — 이러면 실패한다

- 학생이 이해 못 하는 코드가 저장소에 쌓인다 → 버그가 나면 아무도 못 고친다
- 추상화 레이어를 미리 만든다 → 2주짜리 프로젝트에 인터페이스 설계는 사치다
- "일단 동작하게" 하고 설명을 미룬다 → 설명은 영원히 안 온다

**의심스러우면 더 단순하게.**

---

## 2. 무엇을, 왜 만드는가

### 문제

청각장애·난청 학생은 교실에서 소리로 오가는 정보를 통째로 놓친다.

- 여러 명이 말할 때 **누가, 어디서** 말하는지 모른다
- 이름 호명, 종소리, 질문 같은 **말이 아닌 소리**를 놓친다
- 하고 싶은 말이 있어도 **발화가 어려워 참여하지 못한다**

이 프로젝트는 인공와우를 사용하는 실제 당사자의 자문에서 출발했다.
자문에서 가장 강하게 나온 지적은 **"자막보다 누가 말하는지 아는 게 급하다"** 였고,
그래서 화자 방향(DOA) 표시가 자막과 동급의 핵심 기능이 되었다.

### 기존 제품과 무엇이 다른가

시중의 자막 안경·자막 앱은 **클라우드 음성인식에 의존한다.**

|            | 기존 제품                    | 소리봄                                |
| ---------- | ---------------------------- | ------------------------------------- |
| 인터넷     | 필요                         | **불필요 — 완전 오프라인**            |
| 프라이버시 | 교실 음성이 외부 서버로 전송 | 음성이 기기 밖으로 나가지 않음        |
| 화자 방향  | 없음                         | **4채널 마이크로 DOA 추정**           |
| 방향성     | 듣기만 (한 방향)             | **양방향 — 타이핑하면 기기가 말한다** |

**"완전 오프라인"은 이 프로젝트의 정체성이다.** 편의를 이유로 클라우드 API(OpenAI Whisper API,
Google STT 등)를 제안하지 말 것. 그 순간 이 작품의 존재 이유가 사라진다.

## 3. 범위 — 지금 만드는 것과 만들지 않는 것

전체 구상은 여섯 기능이지만, **개발 기간이 2주뿐이라 세 개로 좁혔다.**

### 만든다 (3개)

|     | 기능              | 핵심                                               |
| --- | ----------------- | -------------------------------------------------- |
| ①   | **실시간 자막**   | 2-패스: 빠른 임시 자막 → 정확한 교정 자막          |
| ②   | **누가·어디서**   | 4채널 마이크의 DOA 값을 화면에 방향으로 표시       |
| ⑥   | **양방향 말하기** | 학생이 타이핑 → 단말 내장 스피커가 대신 발화 (TTS) |

그리고 이 셋을 **한 화면(UI)으로 통합**한다.

### 만들지 않는다 (아래 셋은 다음 단계)

- ③ 비음성 소리 이벤트 분류 (이름·종·질문 감지)
- ④ 블루투스로 깨끗한 소리 전달 (AGC + 세이프티 리미터)
- ⑤ 수업 요약 노트 (추출식 요약)

**중요:** `src/` 에 이 셋의 스텁 파일이 이미 있다 (`events/`, `summary/`).
**지금은 구현하지 말 것.** 스텁은 나중을 위한 자리다. 시간이 남으면 그때 논의한다.

시간이 남으면 새 기능을 추가하는 대신 **③ 세 기능을 더 안정적으로** 만든다.

## 4. 이미 확정된 설계와 그 이유

되돌리지 말 것. 각 결정에는 이유가 있다.

### 완전 오프라인 · 엣지 연산

모든 처리를 NVIDIA Jetson Orin Nano 안에서 한다. 네트워크 의존성 0.

### 2-패스 음성인식

```
음성 구간 검출 (Silero VAD)
  → 1패스: small 모델 → 임시 자막 (흐리게 표시, ~0.5초)
  → 2패스: medium 모델 → 교정 자막 (선명하게 교체, ~2초)
```

**왜:** 큰 모델은 정확하지만 느리고, 작은 모델은 빠르지만 부정확하다.
둘 중 하나를 고르는 대신 **작은 모델이 먼저 띄우고 큰 모델이 뒤따라 고친다.**
사용자는 즉시성과 정확도를 모두 얻는다. 이게 이 작품의 기술적 핵심 중 하나다.

### 화자 방향(DOA)은 마이크 칩이 계산한다

ReSpeaker Mic Array v3.0의 **XMOS XVF-3000 칩이 빔포밍·DOA·에코제거를 하드웨어에서 처리**한다.
Jetson은 USB로 **각도 값만 읽으면 된다.**

**직접 DOA 알고리즘(GCC-PHAT 등)을 구현하지 말 것.** 이미 칩이 하고 있다.
Jetson의 연산은 음성인식에 써야 한다.

### 스피커는 교실이 아니라 단말에 내장한다

TTS 음성을 교실 벽 스피커로 내보내면 반 친구들이 **누가 말했는지 알 수 없다.**
학생 자리의 단말에서 소리가 나오면 **소리의 방향으로 화자를 자연스럽게 인식**한다.
이건 UX 설계 결정이지 기술적 편의가 아니다.

구현: MAX98357A (I2S Class-D 앰프) + 4Ω 스피커 → Jetson 40핀 GPIO의 I2S에 직결.

### TTS 엔진은 MeloTTS (Piper 아님)

원래 Piper 를 쓰기로 했으나, **Piper 는 한국어 음성 모델이 존재하지 않는다.**
(2026-07 확인: https://github.com/rhasspy/piper/discussions/680)

그래서 **MeloTTS** 로 교체했다. 한국어 공식 지원, MIT 라이선스, CPU 실시간 합성.

**Piper 로 되돌리자고 제안하지 말 것.** 한국어가 안 나온다.
MeloTTS 설치가 막히면 `espeak-ng` 를 임시 대안으로 쓴다 (음질은 나쁘지만 한국어가 나온다).

### 요약은 추출식(extractive)으로 (다음 단계)

생성식 요약은 없는 말을 지어낼 수 있어 학습 자료로 신뢰할 수 없다.

## 5. 기술 스택

| 영역           | 선택                           | 비고                                           |
| -------------- | ------------------------------ | ---------------------------------------------- |
| Python         | **3.11**                       | 3.13 금지 — faster-whisper/torch가 아직 미지원 |
| 음성인식       | `faster-whisper` (CTranslate2) | int8 양자화. GPU 있으면 cuda, 없으면 CPU       |
| 음성 구간 검출 | `silero-vad`                   |                                                |
| 음성 합성      | `MeloTTS`                      | 오프라인 한국어 TTS (MIT). **Piper는 한국어 미지원이라 교체** |
| 오디오 I/O     | `sounddevice`, `numpy`         |                                                |
| DOA 읽기       | `pyusb`                        | ReSpeaker의 USB 튜닝 파라미터(DOAANGLE)        |
| UI             | `PySide6`                      |                                                |
| 설정           | `pyyaml` → `src/config.yaml`   |                                                |

**패키지를 추가하기 전에 반드시 상의할 것.** 의존성이 늘수록 Jetson 이식이 어려워진다.

## 6. 하드웨어

| 부품   | 모델                                                  | 연결                        |
| ------ | ----------------------------------------------------- | --------------------------- |
| 연산   | NVIDIA Jetson Orin Nano **Super** Developer Kit (8GB) | —                           |
| 마이크 | ReSpeaker Mic Array v3.0 (XMOS XVF-3000, 4채널)       | USB                         |
| 화면   | 15.6" FHD 터치 모니터                                 | **DisplayPort → HDMI 젠더** |
| 저장   | NVMe SSD 500GB (M.2)                                  | 내장                        |
| 스피커 | MAX98357A 앰프 + 4Ω 5W 스피커                         | **I2S (40핀 GPIO)**         |

### 알아둘 함정

- **Jetson에는 HDMI 단자가 없다.** DisplayPort 1.2 뿐이다. USB-C 영상 출력도 지원하지 않는다.
  모니터 연결에는 DP→HDMI 젠더가 반드시 필요하다. (DP++라 패시브 젠더로 충분)
- **터치는 별도 USB 연결**이 필요하다. HDMI는 영상만 나른다.
- SSD에 방열판이 달려 있으면 케이스에 안 들어간다. 방열판을 떼고 장착한다.

### I2S 배선

| MAX98357A | Jetson 40핀       |
| --------- | ----------------- |
| VIN       | 5V (Pin 2 또는 4) |
| GND       | GND (Pin 6)       |
| BCLK      | I2S_BCLK          |
| LRC       | I2S_LRCLK         |
| DIN       | I2S_SDOUT         |

`sudo /opt/nvidia/jetson-io/jetson-io.py` 로 I2S를 활성화하고 재부팅해야 사운드카드로 잡힌다.
`aplay -l` 로 카드 번호를 확인해 `config.yaml` 의 `tts.output_device` 에 적는다.

## 7. 저장소 구조

```
src/
  main.py           진입점 — 4개 레인을 병렬로 돌린다
  config.yaml       모든 설정은 여기에. 코드에 상수를 박지 말 것
  audio/
    capture.py      4채널 마이크 캡처
    vad.py          Silero VAD — 말하는 구간만 잘라낸다
    doa.py          ReSpeaker에서 방향(각도) 읽기
  stt/
    transcriber.py  2-패스 전사 (draft → refine)
  tts/
    speaker.py      MeloTTS 합성 → I2S 스피커 재생
  ui/
    app.py          자막 + 방향 + 입력창
  events/           ← 이번엔 구현하지 않음
  summary/          ← 이번엔 구현하지 않음

experiments/
  analysis.py       대응표본 t-검정 · 신뢰구간 · 효과크기
  data/             실험 원자료 — 커밋 금지 (.gitignore)

docs/               설계 문서. 결정이 바뀌면 여기도 고칠 것
```

각 모듈에는 **인터페이스와 TODO만 있는 스텁**이 들어 있다. 이걸 채우는 게 작업이다.

## 8. 코드가 아닌 산출물 — 반드시 측정할 것

이 프로젝트는 "동작한다"로 끝나지 않는다. **숫자로 증명**해야 한다.
아래 지표를 측정하는 스크립트도 코드의 일부다. 나중에 몰아서 하지 말 것.

| 지표                  | 무엇을 재는가                                    |
| --------------------- | ------------------------------------------------ |
| **WER** (단어 오류율) | 자막 정확도. 조용할 때 / 시끄러울 때 각각        |
| **자막 지연**         | 말이 끝나고 자막이 뜰 때까지. 1패스 / 2패스 각각 |
| **DOA 오차**          | 실제 방향과 추정 방향의 차이 (도 단위)           |
| **TTS 지연**          | 타이핑 완료 → 스피커에서 소리가 날 때까지        |

기능을 구현할 때 **측정 코드를 같이** 만든다. 나중에 붙이려면 구조를 뜯어야 한다.

## 9. 절대 하지 말 것

- **클라우드 API 사용** — 이 작품의 존재 이유가 사라진다
- **개인정보 커밋** — 이름·주소·연락처가 든 파일, 실험 참가자 데이터
- **출처 없는 코드 차용** — 오픈소스는 쓰되 어디서 왔는지 반드시 명기한다.
  출처 미표기는 규정 위반이다. `README` 또는 코드 주석에 링크를 남긴다
- **학생이 이해 못 하는 코드 작성** — 1장을 다시 읽을 것

## 10. 완료의 정의

세 기능이 **한 화면에서 동시에** 동작하고, 위 4개 지표가 측정되어 있고,
2분짜리 시연 영상에 담을 수 있는 상태.

화려한 코드가 아니라 **카메라 앞에서 확실히 동작하는 것**이 목표다.
