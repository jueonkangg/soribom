# 2. 시스템 설계

![시스템 플로우](figures/system-flow.png)

## 설계 원칙

1. **완전 오프라인** — 모든 연산을 Jetson 안에서. 네트워크 의존성 0.
2. **지연과 정확도의 동시 해결** — 2-패스 전사로 두 마리 토끼를 잡는다.
3. **병렬 처리** — 수음된 오디오를 4개 레인으로 동시에 흘린다.

## 파이프라인

교실 소리 → 4채널 마이크 어레이(빔포밍 + DOA) → Jetson에서 4개 레인 병렬 처리

### 레인 A — 실시간 자막
```
음성 구간 검출 (Silero VAD)
  → faster-whisper 2-패스
      1패스: small 모델 → 임시 자막 (흐리게 표시, ~0.5초)
      2패스: medium 모델 → 교정 자막 (선명하게 교체, ~2초)
  → 화면: 자막 + 화자 라벨
```

**왜 2-패스인가.** 큰 모델은 정확하지만 느리고, 작은 모델은 빠르지만 부정확합니다.
둘 중 하나를 고르는 대신, 작은 모델이 먼저 임시 자막을 띄우고 큰 모델이 뒤따라 교정합니다.
사용자는 "즉시 뜨는 자막"과 "정확한 자막"을 모두 얻습니다.

### 레인 B — 누가·어디서
```
DOA(도래각) 추정 → 화자 위치 매핑 → 화면에 방향 표시
```
XMOS XVF-3000 칩이 DOA를 하드웨어에서 계산해 USB로 각도를 넘겨줍니다.

### 레인 C — 소리 알림
```
비음성 오디오 이벤트 분류 → 이름/종/경보/질문 판별 → 시각·진동 알림
```

### 레인 D — 깨끗한 소리 전달
```
근접 강화 오디오 → AGC + 세이프티 리미터 → 블루투스 송출 → 이어버드/보청기
```
데모용 출력이 일반 이어버드(자체 음량 보호 없음)이므로 **세이프티 리미터는 필수**입니다.

### 양방향 말하기 (TTS)
```
학생 타이핑 입력 → MeloTTS (오프라인) → I2S → 단말 내장 스피커
```

> **왜 MeloTTS 인가:** 원래 Piper 를 쓰려 했으나, 조사 결과
> [Piper 는 한국어 음성 모델이 없습니다](https://github.com/rhasspy/piper/discussions/680).
> MeloTTS 는 한국어를 공식 지원하고 MIT 라이선스이며 CPU 만으로 실시간 합성이 가능합니다.
스피커를 **단말에 내장**한 것이 핵심 설계 결정입니다. 교실 스피커에서 소리가 나면
반 친구들이 누가 말했는지 알 수 없지만, 학생 자리에서 나면 방향으로 화자를 인식합니다.

### 수업 종료 후 (배치)
```
누적 전사 + 화자 데이터 → 추출식 요약 → 화자별 요약 노트 저장 (SSD)
```
생성식(generative) 요약이 아닌 **추출식(extractive)** 요약을 씁니다.
없는 말을 지어내지 않으므로 학습 자료로서 신뢰할 수 있습니다.

## 화면 (UI)

![UI 목업](figures/ui-mockup.png)

자막 + 화자 방향 + 소리 알림 + 말하기 입력을 한 화면에 배치했습니다.

## 용어

| 용어 | 뜻 |
|---|---|
| **STT** | Speech-to-Text. 말소리를 글자로 바꾸는 기술 |
| **TTS** | Text-to-Speech. 글자를 말소리로 바꾸는 기술 |
| **VAD** | Voice Activity Detection. 소리에서 '사람이 말하는 구간'만 골라내는 기술 |
| **DOA** | Direction of Arrival. 소리가 어느 방향에서 왔는지 추정하는 기술 |
| **빔포밍** | 여러 마이크의 시간차를 이용해 특정 방향 소리만 키우는 기술 |
| **AGC** | Automatic Gain Control. 소리 크기를 자동으로 알맞게 조절 |
| **엣지 컴퓨팅** | 서버로 보내지 않고 기기(엣지)에서 직접 연산하는 방식 |
