# 1부. Jetson 부팅하기 — 강주언 (+ 막히면 전원)

⏱️ 예상 1~2시간 (이미지 굽는 데만 30~40분)

## 왜 하나요

Jetson은 우리 기기의 "두뇌"입니다. 지금은 아무것도 안 깔린 빈 컴퓨터라서,
**운영체제(JetPack)를 SD카드에 구워서 넣어줘야** 켜집니다.
휴대폰을 처음 사서 초기 설정하는 것과 같습니다.

## 1-1. JetPack 이미지 내려받기

1. [developer.nvidia.com/embedded/jetpack](https://developer.nvidia.com/embedded/jetpack) 접속
2. **Jetson Orin Nano Developer Kit**용 **SD Card Image** 다운로드
3. 파일이 큽니다 (약 10GB). **와이파이 좋은 곳에서** 받으세요
4. 받은 파일은 `.zip` 입니다. **압축을 풀지 마세요.** 굽는 프로그램이 알아서 합니다

> 💡 다운로드가 오래 걸리니, **이걸 먼저 걸어두고** 아래 1-2를 준비하세요.

## 1-2. balenaEtcher 설치

SD카드에 이미지를 굽는 프로그램입니다.

```bash
brew install --cask balenaetcher
```

또는 [etcher.balena.io](https://etcher.balena.io/)에서 직접 받아 설치.

## 1-3. microSD에 굽기

1. microSD를 맥에 꽂습니다 (리더기가 필요할 수 있음)
2. balenaEtcher 실행
3. **Flash from file** → 아까 받은 `.zip` 선택
4. **Select target** → microSD 선택

> ⚠️ **여기서 잘못 고르면 맥 하드디스크가 지워집니다.**
> 용량을 보고 **64GB짜리가 맞는지** 두 번 확인하세요.

5. **Flash!** 클릭 → 맥 비밀번호 입력
6. **20~30분 걸립니다.** 기다리세요. 중간에 뽑지 마세요
7. 끝나면 "이 디스크를 읽을 수 없습니다" 같은 맥 경고창이 뜰 수 있는데 **정상입니다.** 무시하고 꺼내기

## 1-4. 선 연결하기

전원은 **맨 마지막에** 꽂습니다. 순서대로 하세요.

1. **microSD**를 Jetson 바닥면 슬롯에 꽂기 (딸깍 소리)
2. **모니터 연결**
   - Jetson의 **DisplayPort** 단자에 → **DP→HDMI 젠더** 꽂기
   - 젠더에 → HDMI 케이블 → 모니터

> ⚠️ Jetson에는 **HDMI 단자가 없습니다.** DisplayPort만 있어요.
> 젠더 없이는 화면이 절대 안 나옵니다.

3. **모니터 전원** — USB-C 충전기를 모니터에 연결
4. **키보드·마우스**를 Jetson USB 포트에
5. **마지막에 Jetson 전원 어댑터** 연결

## 1-5. 첫 부팅

1. 화면에 NVIDIA 로고가 뜨면 성공입니다 🎉
2. 초기 설정을 따라갑니다
   - 언어: **English 권장** (한글로 하면 나중에 터미널 경로에서 문제가 생길 수 있음)
   - 계정 이름·비밀번호: **팀이 공유할 것으로 정하고 메모**
   - 와이파이 연결
3. 바탕화면이 나오면 완료

## 1-6. 잘 됐는지 확인

터미널을 열고 (`Ctrl + Alt + T`):

```bash
# SSD가 인식되는지
lsblk

# 모니터링 도구 설치
sudo apt update
sudo apt install -y python3-pip
sudo pip3 install jetson-stats
sudo reboot

# 재부팅 후
jtop
```

`jtop`에서 CPU/GPU 그래프가 보이면 **완벽합니다.** `q`로 나옵니다.

## ⚠️ 안 될 때

| 증상 | 해결 |
|---|---|
| 화면이 아예 안 나옴 | DP→HDMI 젠더가 제대로 꽂혔는지 확인. 모니터 입력을 HDMI로 전환 |
| NVIDIA 로고 후 멈춤 | SD카드 굽기 실패. 다시 굽기 |
| SSD가 `lsblk`에 안 보임 | 방열판 때문에 안 꽂혔을 수 있음 → **방열판 떼고 다시 장착** |
| 전원이 안 켜짐 | 어댑터가 Jetson 전용(19V)인지 확인. 모니터용 USB-C 아님 |

**30분 넘게 막히면 혼자 붙들지 말고 팀 대화방에 사진과 함께 올리세요.**

# 2부. 마이크 인식 확인 — 함도유

⏱️ 예상 15분

## 왜 하나요

우리 마이크(ReSpeaker)는 **드라이버 설치가 필요 없습니다.** 맥과 리눅스는 꽂으면 바로 인식돼요.
정말 그런지 5초 녹음해서 확인만 하면 됩니다.

## 2-1. 꽂고 확인

마이크를 맥 USB에 꽂고 터미널에서:

```bash
system_profiler SPAudioDataType | grep -A 3 -i respeaker
```

`ReSpeaker` 또는 `SEEED` 같은 이름이 보이면 인식된 겁니다.

## 2-2. 실제로 녹음해보기

```bash
brew install sox        # 처음 한 번만
rec test.wav trim 0 5   # 5초 녹음 — 뭔가 말해보세요
play test.wav           # 재생
```

내 목소리가 들리면 **성공**입니다.

## 2-3. 방향(DOA) 읽기 — 도전 과제

우리 제품의 핵심 기능인 "누가·어디서"는 이 마이크의 **방향 각도**를 읽어서 만듭니다.

```bash
cd ~/Desktop
git clone https://github.com/respeaker/usb_4_mic_array.git
cd usb_4_mic_array
pip install pyusb
python tuning.py DOAANGLE
```

숫자(0~359)가 나오면 **오늘 목표 초과 달성**입니다.
손뼉을 방향 바꿔가며 쳐보세요. 숫자가 따라 바뀌면 방향 감지가 되고 있는 겁니다.

> 안 되면 오늘은 넘어가도 됩니다. 내일 Claude Code와 같이 해결하세요.
