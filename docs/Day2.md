## 1. 강구현 — 자막을 "계속" 나오게 만들기

⏱️ 3~4시간

어제는 **파일 하나**를 인식했습니다. 오늘은 **말하는 동안 계속** 나오게 합니다.

### 순서

1. `sounddevice` 로 마이크에서 소리를 계속 받는다
2. **Silero VAD** 로 "말하는 구간"만 잘라낸다
3. 잘라낸 구간을 `faster-whisper` 에 넘긴다
4. 터미널에 자막이 계속 찍힌다

### 왜 VAD가 필요한가

조용한 구간까지 전부 인식하면 **연산이 낭비되고 엉뚱한 글자가 나옵니다.**
사람이 말할 때만 골라내는 것이 VAD입니다.

### 오늘의 성공 기준

> 말을 하면 **몇 초 뒤 터미널에 그 말이 뜬다.** 계속. 멈추지 않고.

### 반드시 같이 할 것

**모델을 한 번만 불러오고 재사용하세요.** 매번 다시 부르면 5초씩 낭비됩니다.

그리고 **자막 지연을 재는 코드를 지금 넣으세요.**

```python
# 말이 끝난 시각 → 자막이 뜬 시각
latency = time.time() - speech_end_time
```

CLAUDE.md 8장의 필수 지표입니다. **나중에 붙이려면 구조를 뜯어야 합니다.**

## 2. 함도유 — 화면 만들기

⏱️ 3~4시간

### 순서

1. `PySide6` 로 창을 하나 띄운다
2. 큰 글씨로 자막이 들어갈 자리를 만든다
3. **방향을 나타내는 화살표**를 그린다
4. `tuning.py` 에서 읽은 각도로 화살표를 돌린다

### 변환식 (어제 측정한 값)

```python
screen_angle = (90 - mic_angle) % 360
```

자세한 내용은 [13-doa-calibration.md](13-doa-calibration.md) 참고.

### 꼭 지킬 것

**`is_voice()` 가 참일 때만 화살표를 갱신하세요.**
조용할 때 값은 직전 방향이 남아 있는 것이라 화살표가 제멋대로 돕니다.

### 오늘의 성공 기준

> 창이 뜨고, **내가 말하는 방향으로 화살표가 돈다.**

자막은 아직 가짜 글씨("테스트")여도 됩니다. 연결은 내일 합니다.

I2S 배선은 나중에 합니다. 오늘은 **소리 파일이 만들어지는지**만 확인합니다.

### 순서

1. MeloTTS 설치를 시도한다
2. 한국어 문장 → `.wav` 파일
3. 그 파일이 실제로 소리가 나는지 확인

### 막히면

MeloTTS 설치가 오래 걸리거나 실패하면 **바로 `espeak-ng` 로 넘어가세요.**
어제 이미 깔았고, 음질은 나쁘지만 한국어가 나옵니다.

```bash
espeak-ng -v ko "안녕하세요. 질문이 있습니다." -w out.wav
```

**오늘 목표는 "좋은 목소리"가 아니라 "소리가 난다"입니다.**

### 오늘의 성공 기준

> 타이핑한 글자가 `.wav` 파일이 되고, 재생하면 들린다.

### 시간이 남으면

TTS 지연을 재는 코드를 넣으세요 — 타이핑 완료 → 소리 시작까지.

# 오늘 꼭 이야기할 것

## 발표에서 쓸 "우리만의 발견"

어제 이런 것이 나왔습니다.

```
small 모델        → 과막성  ❌
medium 모델(1.5GB) → 광학성  ❌
small + 단어 4개   → 광합성  ✅
```

**모델을 키우는 것보다 교실 단어를 미리 알려주는 게 효과적이었습니다.**

이건 인터넷에서 베낀 게 아니라 **우리가 직접 재서 알아낸 것**입니다.
심사에서 "작품 이해도"와 "독창성"에 그대로 들어갑니다.

**이런 발견을 앞으로도 기록하세요.** 실패한 시도까지 남기세요.
