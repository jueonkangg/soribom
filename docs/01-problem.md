# 1. 문제 정의

## 배경

교실은 소리로 굴러가는 공간입니다. 선생님의 설명, 친구의 질문, 이름을 부르는 소리,
수업 종, 짝꿍이 건네는 말. 잘 들리는 학생에게는 배경처럼 흘러가는 이 정보들이,
청각장애·난청 학생에게는 통째로 빠집니다.

## 관찰한 어려움

1. **화자 분간** — 여러 명이 말할 때 누가, 어디서 말하는지 알 수 없다.
2. **비음성 정보 누락** — 이름 호명, 종소리, 경보음처럼 말이 아닌 소리를 놓친다.
3. **실시간 추적과 복습** — 수업을 따라가기도 벅차고, 끝나고 되짚을 자료도 없다.
4. **발화의 어려움** — 하고 싶은 말이 있어도 소리 내어 말하기 어려워 참여에서 밀려난다.

## 기존 해법의 한계

시중의 자막 안경·자막 앱은 대부분 **클라우드 음성인식**에 의존합니다.

- 교실의 모든 대화가 외부 서버로 전송된다 → 학생·교사의 프라이버시 문제
- 학교 와이파이가 끊기면 기능이 멈춘다
- 자막만 줄 뿐, "누가·어디서" 말하는지는 알려주지 않는다
- 학생이 **말할 수 있게** 도와주지는 않는다 (한 방향)

## 소리봄의 차별점

| | 기존 제품 | 소리봄 |
|---|---|---|
| 인터넷 | 필요 | **불필요 (완전 오프라인)** |
| 프라이버시 | 음성이 외부로 전송 | 음성이 기기 밖으로 나가지 않음 |
| 화자 방향 | 없음 | **4채널 마이크로 DOA 추정** |
| 비음성 소리 | 없음 | **이름·종·질문 감지 알림** |
| 방향성 | 듣기만 (한 방향) | **양방향 (타이핑 → 발화)** |

## 설계에 반영한 실제 경험

인공와우를 사용하는 청각장애인의 자문을 받아 설계했습니다. 자문에서 나온
가장 중요한 지적은 "자막보다 **누가 말하는지 아는 것**이 급하다"는 것이었고,
이 때문에 화자 방향(DOA) 표시를 핵심 기능으로 올렸습니다.
