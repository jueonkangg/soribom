# DOA 방향 보정 — 실측 기록

**측정일:** 2026-07-19 · **장비:** ReSpeaker Mic Array v3.0 + Jetson Orin Nano

---

## 결과

마이크를 책상에 고정하고, 네 방향에서 5초간 목소리를 냈다.
목소리가 감지된 순간의 각도만 모아 중앙값을 냈다 (표본 각 54~58개).

| 위치 | 중앙값 | 범위 | 표본 |
|---|---|---|---|
| 앞 | **90°** | 84 ~ 91 | 54 |
| 왼쪽 | **172°** | 172 ~ 174 | 58 |
| 뒤 | **272°** | 271 ~ 277 | 56 |
| 오른쪽 | **346°** | 322 ~ 355 | 54 |

## 좌표계

각도는 **오른쪽이 0°, 반시계 방향으로 증가**한다.
수학의 좌표평면과 같은 규칙이다.

```
        앞 90°
          │
 왼쪽 ────┼──── 오른쪽 0°
  180°    │
        뒤 270°
```

## 화면 표시용 변환

화면에는 **앞이 0°, 시계 방향**이 자연스럽다.

```python
screen_angle = (90 - mic_angle) % 360
```

| | 마이크 | 화면 |
|---|---|---|
| 앞 | 90 | 0 |
| 오른쪽 | 346 | 104 (≈90) |
| 뒤 | 272 | 178 (≈180) |
| 왼쪽 | 172 | 278 (≈270) |

## 오차

이론값(90도 간격)과 비교한 차이다.

| 위치 | 이론 | 실측 | 차이 |
|---|---|---|---|
| 앞 | 90 | 90 | 0° |
| 왼쪽 | 180 | 172 | 8° |
| 뒤 | 270 | 272 | 2° |
| 오른쪽 | 360 | 346 | 14° |

**평균 오차 6°**

> ⚠️ 이 값에는 **사람이 정확히 90도 위치에 서지 못한 오차**가 섞여 있다.
> 정식 측정에서는 바닥에 각도를 표시하고 재야 한다.

## 측정 방법 — 왜 이렇게 했는가

처음에는 **손뼉을 치고 엔터를 눌러** 값을 읽었다. 결과가 엉망이었다.
같은 위치를 네 번 쟀는데 258 / 178 / 0 / 21 이 나왔다.

원인이 두 가지였다.

**1. 소리가 끝난 뒤에 읽었다.**
손뼉과 엔터 사이에 엔터 치는 소리, 발소리가 들어가 방향을 덮어썼다.
칩은 항상 **가장 최근 소리**의 방향을 들고 있다.

**2. 손뼉은 부적절하다.**
XMOS 칩은 사람 목소리를 기준으로 설계됐다.
순간적인 충격음은 벽 반사로 엉뚱한 방향이 잡힌다.

그래서 **소리를 내는 동안 계속 읽고, 목소리가 감지된 값만** 모으도록 바꿨다.

```python
if mic.is_voice():
    angles.append(mic.direction)
```

`is_voice()` 로 거르는 것이 핵심이다.
조용할 때의 값은 직전 방향이 남아 있는 것이라 의미가 없다.

**이건 측정용 요령이 아니라 실제 제품에 그대로 들어갈 로직이다.**
사람이 말할 때만 화면의 화살표를 갱신해야 한다.

## 측정 시 지킬 것

- 마이크는 **책상에 고정**한다 (손에 들지 않는다)
- 측정하는 사람은 **이동하지 않는다** — 각 위치에 다른 사람이 선다
- 위치마다 **3회 이상** 반복한다
- 범위(min~max)가 ±20° 안이면 신뢰할 수 있다
