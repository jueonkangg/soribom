# 작품요약서 — 소리봄 (SoundSight)

**팀명:** 제주 BTS · **팀원:** 강주언(리더)·함도유·강구현
**분야:** 청각장애·난청 학생을 위한 교실 보조기기 (완전 오프라인 엣지 AI)
**한 줄 정의:** 교실의 소리를 **글자와 방향**으로 보여 주고, 말하기 어려운 학생의 **타이핑을 목소리로** 대신 전하는, **인터넷 없이 동작**하는 단말.

---

## 1. 작품 개요

소리봄은 잘 들리지 않는 학생이 교실 수업에 **온전히 참여**하도록 돕는 기기다. 세 가지 기능을
한 화면에서 동시에 제공한다 — **① 실시간 자막, ② 화자 방향 표시(누가·어디서), ⑥ 양방향 말하기.**
모든 처리는 소형 엣지 컴퓨터(NVIDIA Jetson Orin Nano) 안에서 이뤄지며, **인터넷 연결이 필요 없다.**

## 2. 개발 배경 및 필요성

- 청각장애·난청 학생은 교실에서 **누가·어디서** 말하는지, **말이 아닌 소리(이름 호명·종소리)**를 놓치고,
  발화가 어려워 **참여 자체가 막힌다.** 수업은 되감기가 되지 않아 한 번 놓치면 회복이 어렵다.
- 본 작품은 **인공와우 사용 당사자의 자문**에서 출발했다. 가장 강한 지적은
  **“자막보다 누가 말하는지를 아는 게 더 급하다”**였고, 이에 따라 **화자 방향 표시를 자막과 동급의
  핵심 기능**으로 설계했다.
- 시중의 자막 안경·앱은 대부분 **클라우드 음성인식에 의존**한다. 이는 ①교실 음성이 외부로 전송되어
  **프라이버시 문제**가 있고, ②**인터넷이 불안정한 환경**에서 동작하지 않으며, ③**방향·양방향 기능이
  없다.** 소리봄은 이 한계를 정면으로 넘고자 했다.

## 3. 주요 기능

| 기능 | 내용 |
|---|---|
| **① 실시간 자막** | 발화를 실시간으로 글자화. 도메인 단어 힌트로 교과 용어까지 정확. |
| **② 누가·어디서 (DOA)** | 4채널 마이크로 소리의 방향을 추정해 화면 화살표로 표시. |
| **⑥ 양방향 말하기 (TTS)** | 학생이 타이핑한 문장을 단말 스피커가 또렷한 한국어 음성으로 발화. |
| **통합** | 세 기능을 **한 화면**에서 동시에 구동(자막·방향 화살표·입력창). |

## 4. 시스템 구성 및 사용 기술

- **하드웨어:** NVIDIA Jetson Orin Nano Super(연산·GPU), ReSpeaker 4채널 마이크(XMOS XVF-3000,
  빔포밍·DOA·에코제거를 칩에서 처리), 터치 디스플레이, NVMe SSD, 블루투스/내장 스피커.
  케이스는 재고 부족으로 **3D 프린팅**으로 자체 제작.
- **소프트웨어(전부 오프라인):** 음성인식 `faster-whisper`(small·int8·GPU), 음성구간검출 Silero VAD,
  음성합성 `sherpa-onnx` + **Supertonic v3**(한국어 네이티브, GPU 실시간), UI `PySide6`, DOA는
  ReSpeaker USB 파라미터를 읽어 처리.
- **완전 오프라인 엣지 연산:** 음성이 기기 밖으로 나가지 않는다.

## 5. 차별성 및 독창성

1. **완전 오프라인** — 프라이버시 보장, 인터넷 없는 환경에서도 동작, 낮은 지연.
2. **화자 방향 표시** — 당사자 요구를 반영한 핵심 차별점. 4채널 마이크로 “누가·어디서”를 화살표로.
3. **양방향** — 듣기만 돕는 기존 제품과 달리, 학생이 **말하도록**까지 돕는다.
4. **‘모델보다 단어’ 발견(실측 기반 독창성)** — 큰 모델(medium)은 메모리 초과·정확도 향상 없음을
   실측으로 확인하고, **작은 모델 + 수업 용어 힌트**로 더 싸고 정확하게 해결(문자오류율 0%대).
5. **연산 분업** — 복잡한 방향 계산은 마이크 칩에 위임하고, 연산 자원은 음성인식에 집중.

## 6. 개발 결과 및 성능 (실측)

모든 수치는 실제 기기에서 자체 측정 도구로 측정한 값이다.

| 지표 | 결과 | 조건 |
|---|---|---|
| 자막 오류율(CER) | **0.0% / 0.7%** | 조용할 때 / 시끄러울 때 |
| 자막 지연 | **약 0.6초** | 말이 끝난 뒤 자막까지 |
| 말하기(TTS) 지연 | **약 1.8초** | 타이핑 완료 → 발화 |
| 화자 방향(DOA) 오차 | **평균 1.5°** | 케이스 조립 후, 4방향 평균 |
| 통합 안정성 | **OOM 없음(여유 3.6GB)** | 자막·말하기 GPU 동시 상주 |

- 세 기능이 **한 화면에 함께 구성**되고, 개별 동작·STT+TTS 동시 상주·타이핑→발화 라이브를 확인.
  (마이크→자막+방향의 완전 동시 라이브는 시연 촬영에서 시연)
- 개발 중 **케이스가 마이크를 덮으면 방향이 붕괴**함을 측정으로 발견 → 마이크 면 노출로 해결(1.5°).

## 7. 기대 효과 및 활용 방안

- **교실:** 잘 안 들리는 학생이 대화 흐름·화자·용어를 놓치지 않고 **직접 발언**까지 하며 참여.
- **확장:** 같은 뼈대(오프라인·자막·방향·양방향)를 회의·병원 창구·가정(노년층 난청) 등으로 확장 가능.
- **프라이버시 우선:** 음성이 외부로 나가지 않아 미성년자 환경에 적합.

## 8. 향후 계획

- **다음 단계 세 기능:** ③ 비음성 소리 이벤트 감지(이름·종·질문), ④ 블루투스로 보청기에 깨끗한 소리
  전달(안전 리미터 포함), ⑤ 수업 요약(추출식, 지어내지 않음).
- **성능 강화:** 여러 화자 동시 분리, 배터리형 소형화.
- **하드웨어 마무리:** 최종 케이스에 마이크 개구부 확정.
