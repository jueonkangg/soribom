# 강구현 개발 로그 (STT / 자막 파이프라인)

내가 맡은 부분: **소리를 받아서 → 말하는 구간만 잘라서 → 자막으로 바꾸기**.
어려운 말로는 STT(음성인식) pipeline이다.

---

## 2026년 7월 20일

### 오늘 한 것

**1. 내 작업 공간 만들기 (git branch)**
- `main`은 팀 공용이라 여기서 막 고치면 안 된다. 그래서 내 branch인 `guhyeon`에서 작업했다.
- 커밋에 내 이름이 찍히게 git 설정(`user.name`, `user.email`)도 했다.
- 배운 것: branch는 **각자 따로 노는 놀이터** 같은 거다. 내가 여기서 코드를 망가뜨려도 친구들 것은 안 망가진다. 다 되면 `main`으로 합친다(merge).

**2. 마이크에서 소리 받기 (`capture.py`)**
- ReSpeaker라는 4채널 mic에서 소리를 읽어 `queue`(줄서기 통)에 넣는 코드를 짰다.
- `sounddevice`가 소리를 조금씩 모아서 `callback`이라는 함수를 자동으로 불러준다. 나는 그때마다 소리를 queue에 넣기만 하면 된다.
- 이 mic는 채널이 6개인데, **채널 0번이 칩이 미리 깨끗하게 다듬어 준 소리**라 그것만 쓰기로 했다.
- 확인 방법: 3초 녹음해서 소리 크기(RMS)를 찍어보고 `.wav`로 저장해서 직접 들어봤다.
- 배운 것:
  - `queue` = 소리 데이터를 한 칸씩 넣고 빼는 통. 넣는 쪽(mic)과 쓰는 쪽(VAD)을 이어준다.
  - `callback` = 내가 부르는 게 아니라 **필요할 때 자동으로 불려지는** 함수.
  - 소리는 `float32` 숫자(-1 ~ 1)로 들어온다. RMS가 크면 소리가 큰 거다.

**3. 말하는 구간만 자르기 (`vad.py`)**
- VAD = Voice Activity Detection. **지금 사람이 말하는 중인지 아닌지**를 판단하는 것.
- `silero`라는 똑똑한 VAD 모델을 써서, 소리를 512개 샘플씩(32ms) 잘라 "말일 확률"을 매겼다.
- 확률이 높으면 모으고, 조용해지면 "한 문장 끝!" 하고 그 구간을 내보낸다.
- 처음 만들 때 앞부분이 잘리지 않게 살짝 여유(`speech_pad_ms`)를 앞에 붙였다.
- 결과: 8초 동안 문장 몇 개를 말했더니 **4개 발화로 정확히 잘라냈다.**

### 오늘 크게 막혔던 것 (그리고 어떻게 풀었나)

- 처음엔 인터넷에서 받은 silero 모델을 직접 붙였는데, **말을 해도 확률이 계속 0**이 나왔다. 소리는 분명히 크게 들어오는데(RMS 0.11) 모델만 못 알아들었다.
- 원인을 찾으려고, **이미 잘 작동하는 것(faster-whisper 안에 든 VAD)과 같은 소리로 비교**해봤다. 그건 말을 잘 찾아냈다. 즉 소리는 문제없고 내 모델 부르는 방식이 틀린 거였다.
- 그래서 **faster-whisper에 이미 들어있는 VAD로 갈아탔더니 바로 해결**됐다. 덤으로 무거운 `torch`도 안 깔아도 됐다.
- 배운 것:
  - 안 될 때는 혼자 헤매지 말고 **"되는 것"과 비교**하면 원인이 좁혀진다.
  - 이미 잘 만들어진 걸 쓰면(재사용) 시간도 줄고 버그도 준다.
  - `Jetson`(우리 컴퓨터)에서는 `torch` 깔기가 까다로워서, 되도록 `onnx`로 도는 걸 쓰는 게 좋다.
